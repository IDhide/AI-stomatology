# Smile.AI — голосовой администратор стоматологии

Голосовой ассистент-администратор клиники «Стоматология №1» (персона **Оливия**):
встречает пациента у экрана, отвечает голосом, консультирует по услугам и ценам,
ведёт к записи на бесплатную консультацию. Запись идёт через **тестовый DIKIDI
API** (легко переключается на реальный).

Один рабочий вариант — **веб-киоск**: страница открывается в браузере в
полноэкранном режиме (на ТВ/мини-ПК), визуал — пульсирующий круг с аудио-волной
и медузы в режиме ожидания.

![визуал](docs/ui_speaking.png)

## Что внутри

| Слой        | Реализация                                                        |
|-------------|-------------------------------------------------------------------|
| Визуал      | `web/` — HTML/CSS/JS, круг + аудио-волна + субтитры, фон-медузы    |
| Голос (TTS) | `speechSynthesis` браузера (русский голос)                        |
| Слух (STT)  | Chrome/Edge — Web Speech API; Firefox — сервер (`faster-whisper`)  |
| Мозг        | Ollama LLM (персона из `config/prompts.yaml`) или правила-фолбэк   |
| DIKIDI      | `src/dikidi/client.py` → тестовый `src/dikidi/fake_server.py`      |
| Сервер      | `src/web/server.py` (aiohttp): статика, SSE, /api/*                |

## Быстрый старт (Docker)

```bash
cp .env.example .env

# базовый: голос + тестовый DIKIDI (мозг — правила)
make demo

# умный: + языковая модель Ollama (понимает контекст; качает модель ~1.9 ГБ)
make smart
#   для Firefox добавить серверное распознавание речи:
WITH_STT=1 make smart
```

Открыть **http://localhost:8080** (в Chrome/Edge — распознавание сразу; в
Firefox — с `WITH_STT=1`). Кликнуть по экрану, разрешить микрофон, говорить.

Тестовый DIKIDI API — на **http://localhost:8089** (`/healthz`, `/v1/...`).

Подробности, режимы и запуск без Docker — в **[docs/WEB_DEMO.md](docs/WEB_DEMO.md)**.

## DIKIDI: тест → реальный

- `src/dikidi/fake_server.py` — тестовый сервер API (повторяет пути DIKIDI).
- `src/dikidi/client.py` — клиент, которым пользуется ассистент.

Сейчас клиент ходит в тестовый сервер. Для реального DIKIDI поменяйте в `.env`:

```env
DIKIDI_BASE_URL=https://api.dikidi.net
DIKIDI_TOKEN=ваш_токен
```

Код менять не нужно (возможна правка путей/полей под реальные ответы DIKIDI).

## Как это работает на телевизоре

Сам ТВ обычно не запускает приложение — нужен **мини-ПК / Android-box**,
подключённый к ТВ по HDMI, USB-камера с микрофоном и колонки (или звук по HDMI):

1. На мини-ПК запускается этот сервер (`make demo` / `make smart`).
2. Браузер открывается в полноэкранном (kiosk) режиме на `http://localhost:8080`.
3. Микрофон USB-камеры → распознавание, ответы Оливии → колонки/ТВ.
4. Камера может определять появление человека и сама начинать диалог
   (`?camera=1`).

## Конфигурация

- `config/prompts.yaml` — персона Оливии, стиль, цены, акции, правила (мозг LLM).
- `config/settings.yaml` — данные клиники, тайминги, UI.
- `.env` — `OLLAMA_MODEL`, `DIKIDI_BASE_URL`/`DIKIDI_TOKEN`, `STT_MODEL`.
