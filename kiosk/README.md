# Киоск Оливии — визуальный режим «Океан Оливии»

Полноэкранная процедурная сцена для ТВ-панели ресепшена. Живой подводно-космический
мир (биолюминесцентная глубина, реалистичные медузы, частицы) со скрытым
энергетическим ядром Оливии, которое при появлении посетителя превращается в
аудиореактивную голосовую сферу. Всё генерируется в реальном времени на Three.js +
GLSL — это не видео и не зациклённая заставка.

## Запуск

Статику раздаёт стриминговый сервер (`server/app/main.py`) — он монтирует папку
`kiosk/` в корень. Открыть в Chromium/Electron в режиме киоска:

```
http://<host>:<port>/            # прод (без служебных надписей)
http://<host>:<port>/?debug      # + панель ручного прогона состояний
```

Кнопка «Запустить киоск» нужна один раз, чтобы браузер разрешил звук и микрофон.

## Структура

```
kiosk/
  index.html         точка входа, importmap на локальный Three.js (vendor/)
  app.js             связка: WebSocket ↔ микрофон ↔ плеер ↔ сцена
  audio.js           захват микрофона (VAD) и плеер PCM16 с AnalyserNode (§16)
  visuals.js         реэкспорт сцены (обратная совместимость)
  config.json        параметры сцены и профиль качества (§22)
  vendor/three/      Three.js + addons локально — киоск работает без CDN/сети (§24)
  scene/
    OceanScene.js    оркестратор: слои, машина состояний, bloom, адаптивный FPS
    background.js     анимированный градиент + procedural noise (§6.1)
    lightrays.js      мягкие световые лучи без источника (§6.3)
    particles.js      фоновые частицы, планктон, передний план (§6.4)
    jellyfish.js      процедурные медузы: купол (vertex-деформация) + щупальца (§7)
    core.js           ядро Оливии и активная сфера (§8, §13, §16)
    camera.js         noise-дрейф виртуальной камеры (§9)
    events.js         редкие кинематографические события (§19)
    glsl.js           общий GLSL (simplex noise / fbm)
    config.js         профили качества Low/Medium/High + загрузка config.json
```

## Управление состояниями (извне)

Сцена следует событиям сервера по WebSocket (`app.js` → `OceanScene`):

```json
{ "type": "presence", "value": true }               // человек обнаружен → awakening
{ "type": "state", "value": "listening|thinking|speaking|idle" }
```

Состояния: `idle · awakening · greeting · listening · thinking · speaking ·
goodbye · return_to_idle`. Переходы бесшовные — без смены страницы и чёрного кадра.

Аудиореактивность сферы берётся из реально воспроизводимого TTS через
`AnalyserNode` (RMS + низкие/средние/высокие полосы), а не из имитации.

## Качество и производительность

Профиль (`low` / `medium` / `high`) задаётся в `config.json`; при отсутствии —
подбирается автоматически по устройству. Если FPS устойчиво < 45, рендер один раз
снижает pixelRatio и bloom. Цель — 60 FPS в 1920×1080, стабильная работа 8+ часов
без роста памяти.

## Отказоустойчивость

При потере связи с backend сцена продолжает жить в режиме ожидания. При ошибке
WebGL включается запасной CSS-фон (`#idle-fallback`).
