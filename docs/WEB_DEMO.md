# 🌐 Веб-демо Smile.AI — визуал, камера и симуляция DIKIDI

Этот демо-стенд позволяет посмотреть **как выглядит интерфейс** ассистента и как
система реагирует на появление человека — **без GPU, без Ollama, без физического
дисплея**. Всё открывается в обычном браузере.

Боевой UI (`src/ui/display.py`) работает на pygame и требует подключённого
экрана. Веб-версия повторяет тот же визуал (медузы → пульсирующий круг с
аудио-волной → субтитры) и удобна для разработки и демонстрации.

---

## 1. Установка зависимостей (минимум)

```bash
pip3 install -r requirements-minimal.txt
cp .env.example .env
```

Для веб-демо хватает `aiohttp`, `loguru`, `pydantic`, `pyyaml`, `python-dotenv`.

---

## 2. Запуск веб-киоска (визуал + сценарий)

```bash
python3 -m src.web.server --port 8080
```

Откройте <http://127.0.0.1:8080> — пойдёт авто-сценарий:

```
idle (медузы) → 👤 «лицо обнаружено» → greeting → диалог о кариесе
            → запрос свободных окон у DIKIDI → запись → 👋 уход → idle
```

Режимы и их визуал:

| Режим      | Что видно                                                        |
|------------|-----------------------------------------------------------------|
| `idle`     | Медузы / звёздный фон (ожидание)                                 |
| `greeting` | Круг проявляется, лёгкая «волна»                                 |
| `listening`| Зелёный круг, спокойное «дыхание» волны                          |
| `thinking` | Янтарный круг, мелкая рябь                                       |
| `speaking` | Фиолетовый круг + аудио-волна в такт речи + субтитры             |

Флаги:

```bash
python3 -m src.web.server --port 8080 --no-loop   # без авто-сценария
curl -X POST http://127.0.0.1:8080/api/trigger    # вручную «впустить пациента»
```

Горячие клавиши на странице (для отладки без сервера): `1`–`5` переключают режимы.

## 3. Камера и распознавание людей

«Железный» путь MVP — обычная USB веб-камера → браузер. Включается флагом:

```
http://127.0.0.1:8080/?camera=1
```

`web/camera.js`:
- запрашивает доступ к камере (`getUserMedia`),
- детектирует движение (покадровая разница) и лицо (`FaceDetector API`, если есть),
- при устойчивом обнаружении человека шлёт `POST /api/trigger` → запускается
  приветствие и диалог,
- в углу показывает self-view и статус («камера активна» / «человек обнаружен»).

> В headless-окружении (CI/контейнер без камеры) используйте авто-сценарий или
> ручной триггер — логика присутствия там симулируется на сервере.

Боевая детекция на Python (MediaPipe + OpenCV, RTSP/USB) — в
`src/camera/detector.py`.

## 4. Симуляция DIKIDI API

Есть два уровня подмены DIKIDI:

### a) In-process stub (`src/dikidi/client_stub.py`)
Подменяет клиент прямо в памяти — используется веб-сервером и `main_offline.py`.

### b) HTTP-симулятор (`src/dikidi/fake_server.py`) — «настоящий» API
Поднимает HTTP-сервер с теми же путями, что у боевого DIKIDI, поэтому
**продакшн-клиент `src/dikidi/client.py` работает с ним без изменений**.

```bash
# запустить симулятор API
python3 -m src.dikidi.fake_server --port 8089 --no-auth

# проверить
curl http://127.0.0.1:8089/healthz
curl "http://127.0.0.1:8089/v1/company/services"
```

Эндпоинты:

| Метод | Путь                            | Назначение            |
|-------|---------------------------------|-----------------------|
| GET   | `/healthz`                      | проверка живости      |
| GET   | `/v1/company/services`          | список услуг с ценами |
| GET   | `/v1/company/masters`           | список врачей         |
| GET   | `/v1/booking/available-slots`   | свободные окна        |
| POST  | `/v1/booking/create`            | создать запись        |
| GET   | `/v1/clients/search`            | поиск клиента         |

Чтобы боевой клиент ходил в симулятор, укажите в `.env` / `settings.yaml`:

```env
DIKIDI base_url = http://127.0.0.1:8089
```

## 5. Скриншоты интерфейса (headless)

```bash
npm install playwright          # один раз
node scripts/screenshot_ui.js   # сохранит docs/ui_*.png
```

Скрипт открывает киоск в Chromium и снимает все 4 режима. Примеры —
`docs/ui_idle.png` и `docs/ui_speaking.png`.
