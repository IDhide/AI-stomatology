# Архитектура системы

## Обзор

AI Salon Assistant - это модульная система для автоматизации работы администратора салона красоты с использованием голосового AI.

## Модули

### 1. Core (Ядро)

**Файлы:**
- `src/core/app.py` - Главный класс приложения
- `src/core/config.py` - Управление конфигурацией
- `src/core/conversation_logger.py` - Логирование разговоров

**Ответственность:**
- Управление состояниями приложения
- Координация работы всех модулей
- Обработка жизненного цикла разговора

### 2. Camera (Камера)

**Файлы:**
- `src/camera/detector.py` - Детекция присутствия

**Технологии:**
- OpenCV для захвата видео
- MediaPipe для детекции лиц
- Алгоритм детекции движения

**Функции:**
- Обнаружение человека перед камерой
- Детекция лица
- Детекция движения

### 3. Voice (Голос)

**Файлы:**
- `src/voice/stt.py` - Speech-to-Text
- `src/voice/tts.py` - Text-to-Speech

**Технологии:**
- Whisper для распознавания речи
- Silero TTS для синтеза речи

**Функции:**
- Распознавание русской речи
- Синтез естественной русской речи
- Детекция тишины

### 4. LLM (Языковая модель)

**Файлы:**
- `src/llm/assistant.py` - Интеграция с Ollama

**Технологии:**
- Ollama + OpenClaw
- Контекстное управление разговором

**Функции:**
- Генерация ответов на русском
- Управление историей разговора
- Интеграция с DIKIDI API

### 5. DIKIDI (API интеграция)

**Файлы:**
- `src/dikidi/client.py` - HTTP клиент

**Функции:**
- Получение свободных слотов
- Создание записей
- Поиск клиентов
- Получение услуг и мастеров

### 6. UI (Интерфейс)

**Файлы:**
- `src/ui/display.py` - Визуальный интерфейс

**Технологии:**
- Pygame для отрисовки
- OpenCV для видео

**Функции:**
- Анимированный пульсирующий круг
- Воспроизведение видео с медузами
- Индикация состояния системы

## Поток данных

### Сценарий 1: Приветствие клиента

```
1. Camera.detect_person() → True
2. App.state = DETECTING
3. Подтверждение присутствия (2 сек)
4. App.state = GREETING
5. TTS.speak("Здравствуйте!")
6. UI.set_mode("speaking")
7. App.state = CONVERSATION
```

### Сценарий 2: Разговор

```
1. STT.listen() → "Хочу записаться"
2. LLM.get_response() → запрос к DIKIDI
3. DIKIDI.get_available_slots() → список слотов
4. LLM генерирует ответ с учетом слотов
5. TTS.speak(ответ)
6. UI анимирует круг
7. ConversationLogger.log_message()
```

### Сценарий 3: Прощание

```
1. Camera.detect_person() → False
2. App.state = GOODBYE
3. Пауза 3 секунды
4. UI.set_mode("speaking")
5. TTS.speak("До свидания!")
6. App.state = IDLE
7. UI.set_mode("idle") → видео с медузами
```

## Конфигурация

Система использует многоуровневую конфигурацию:

1. **YAML файл** (`config/settings.yaml`) - основные настройки
2. **ENV переменные** (`.env`) - секреты и переопределения
3. **Pydantic модели** - валидация и типизация

## Логирование

### Системные логи

- Консольный вывод с цветами (loguru)
- Ротация файлов (10MB, 5 бэкапов)
- Уровни: DEBUG, INFO, WARNING, ERROR

### Логи разговоров

- JSONL формат
- Каждая строка = одно событие
- Поля: conversation_id, event, role, content, timestamp

## Безопасность

- API ключи в переменных окружения
- Валидация входных данных через Pydantic
- Таймауты для всех сетевых запросов
- Обработка исключений на всех уровнях

## Масштабируемость

### Текущая архитектура

- Однопоточное приложение
- Асинхронные операции (asyncio)
- Локальная обработка

### Возможные улучшения

- Микросервисная архитектура
- Очереди сообщений (RabbitMQ/Redis)
- Распределенное логирование
- Балансировка нагрузки

## Производительность

### Оптимизации

- Квантизация модели LLM (Q4)
- Кэширование ответов DIKIDI
- Предзагрузка моделей
- GPU ускорение (CUDA)

### Метрики

- Время отклика: < 2 сек
- Точность STT: > 90%
- Использование GPU: < 10GB
- CPU: < 50%

## Зависимости

### Критические

- Python 3.10+
- CUDA 11.8+ (для GPU)
- Ollama
- DIKIDI API

### Опциональные

- Камера (можно отключить)
- GPU (можно использовать CPU)
